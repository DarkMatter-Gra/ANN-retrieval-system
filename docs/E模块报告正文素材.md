# E 模块报告正文素材

本文档用于后续填写 `report.tex` 正文，可按需复制到“详细设计”“编码实现”“测试与验证”“总结”章节。

## 1. 模块定位

E 模块负责系统中的查询检索服务与测试文档演示，位于数据预处理模块、ANN 索引模块和前端可视化模块之间。其核心作用是把已经入库的细胞向量、细胞元数据和 ANN 索引组织成可供用户查询、过滤、导出、可视化和诊断分析的后端服务。

从系统调用关系看，E 模块主要消费 C 模块产生的 PCA/UMAP/t-SNE 结果和 D 模块构建的 ANN 索引，并通过 B 模块提供的 FastAPI 接口向前端和测试脚本开放能力。

## 2. 主要功能实现

E 模块实现了以下功能：

- 单细胞 Top-K 检索：支持按 `cell_id` 查询，也支持直接传入向量查询。
- 检索模式切换：支持 `mode=ann` 的近似检索和 `mode=exact` 的精确检索。
- 条件过滤：支持按 `cell_type`、`organ`、`sample_id` 以及 `obs_ext` 扩展字段过滤。
- 批量检索：支持多个查询请求组成批量任务，任务完成后可导出 JSONL 和 CSV 文件。
- 可视化高亮：检索后保存 query cell 和 neighbor cells 的 embedding 坐标，供前端在 UMAP 图上高亮。
- 指标统计：检索任务落库后，可统计最近查询延迟、p50、p95、p99、QPS 和索引状态。
- 诊断报告：支持生成 JSON 报告和 PDF 报告，并通过受权限保护的文件接口下载。
- 测试工具：提供 smoke 测试、真实数据端到端测试和 ANN 参数 benchmark 脚本。

## 3. ANN 引擎设计

项目原有 ANN 逻辑分散在索引构建、索引加载和检索服务中。当前实现新增了统一 `ANNEngine`，将索引参数校验、向量矩阵检查、索引构建、索引加载、查询执行和结果距离语义统一封装。

当前支持的索引类型包括：

- `flat`：精确检索，适合作为小数据集方案和召回率评估 baseline。
- `hnsw`：图索引近似检索，适合中等规模高维向量检索，当前真实数据测试中表现最好。
- `ivf_pq`：倒排文件和乘积量化索引，适合后续更大规模数据压缩检索。

当前支持的距离度量包括：

- `l2`
- `ip`
- `cosine`

为了保证索引结果和业务细胞编号稳定对应，索引构建后会同时生成：

- `.index`：底层 ANN 索引文件
- `.id_map.json`：索引内部位置到 `cell_id` 的映射
- `.meta.json`：索引参数、维度、向量数、构建耗时等元数据

## 4. 过滤检索策略

对于无过滤条件的请求，系统直接在目标 ANN 索引上执行 Top-K 检索。对于带过滤条件的请求，系统先从 ANN 索引中获取扩大后的候选集合，再根据细胞元数据过滤候选结果。如果过滤后候选不足，系统会回退到过滤子集上的精确检索。

这种策略的优点是：

- 保证返回结果满足过滤条件
- 在大多数普通查询中仍能利用 ANN 加速
- 过滤命中较少时仍能保证结果正确性

后续若数据规模继续扩大，可以进一步升级为 bitmap 预过滤或按高频标签构建子索引。

## 6. ANN 性能基准结果

为了量化 ANN 算法效果，系统使用 Flat 精确检索作为 baseline，对 HNSW 多组参数进行了 benchmark。测试设置如下：

| 项目 | 数值 |
|---|---:|
| 数据集 | `liver_real_test` |
| 查询数 | 200 |
| Top-K | 10 |
| 距离度量 | L2 |
| 随机种子 | 42 |

关键结果：

| 方法 | 参数 | ef_search | recall@10 | p95 延迟(ms) |
|---|---|---:|---:|---:|
| Flat 精确检索 | `{}` | - | 1.0000 | 4.6090 |
| HNSW Fast | `M=8, ef_construction=80` | 64 | 0.9925 | 0.0255 |
| HNSW Balanced | `M=16, ef_construction=120` | 128 | 0.9985 | 0.0693 |
| HNSW Quality | `M=32, ef_construction=200` | 256 | 1.0000 | 0.1345 |

基于该结果，当前项目推荐将 HNSW 作为默认 ANN 索引方案。如果演示更强调准确性，可使用：

```json
{
  "M": 32,
  "ef_construction": 200,
  "ef": 256
}
```

如果更强调速度和构建成本，可使用：

```json
{
  "M": 16,
  "ef_construction": 120,
  "ef": 128
}
```

## 8. 小结

E 模块已经完成课程分工中的查询检索服务和测试文档演示任务。系统不仅可以在小型测试数据上通过自动化测试，也已经在老师提供的真实单细胞数据集上跑通了预处理、索引构建、检索、过滤、导出、可视化高亮、指标和诊断报告全链路。通过 ANN benchmark，当前 HNSW 引擎在真实数据上达到较高召回率和低延迟，能够支撑课程系统的检索演示和性能分析。
