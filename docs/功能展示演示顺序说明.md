## 4. 推荐录屏主线

### 4.1 登录与工作台

1. 打开 `http://localhost:4173/login`

4. 简要展示左侧角色化导航和首页统计入口

可说明：系统支持多角色用户，不同角色有不同页面权限。

### 4.2 数据集管理

进入“数据集管理”。

展示重点：

- 数据集名称：`liver`
- 文件格式：`h5ad`
- 细胞数：`30000`
- 基因数：`32397`
- 预处理状态：`done`
- 质控状态：`passed`

建议点击：

1. 选择 `dataset_id = 6`
2. 查看数据集详情
3. 查看预处理日志
4. 展示上传区域支持 `.h5ad / .mtx / .csv`

说明口径：本次演示使用老师提供的 `liver.h5ad`，该文件约 1.4GB。真实上传和预处理链路已验证可用，录屏时建议展示上传入口，不建议现场等待完整重新上传。

### 4.3 索引管理

进入“索引管理”。

推荐展示：

```text
dataset_id: 6
index_id: 1
index_type: flat
metric: l2
build_status: done
recall_score: 1.0
memory_cost_mb: 3.4332
```

操作顺序：

1. 点击“刷新”或“查询”
2. 选择 `index_id = 1`
3. 点击“详情”
4. 点击“加载到内存”

可说明：索引构建基于预处理后的 PCA 向量，E/D 模块不直接读取原始 h5ad，而是消费 C/B 模块入库后的向量。

### 4.4 单次 ANN 检索

进入“向量检索”。

填写：

```text
数据集 ID: 6
索引 ID: 1
查询类型: cell_id
cell_id: AAACCTGAGCAGGTCA-1_2
Top K: 5
模式: ann
度量: l2
filters JSON: {}
```

点击“执行检索”。

预期：

- 返回 Top-K 近邻列表
- Top1 是查询细胞本身
- 页面显示 `query_id`
- 页面显示延迟、距离、分数、细胞类型、器官、样本 ID

本次测试中 Top1：

```text
AAACCTGAGCAGGTCA-1_2
```

### 4.5 筛选检索

仍在“向量检索”页面，把 filters 改成：

```json
{"cell_type":["hepatocyte"]}
```

点击“执行检索”。

预期：

- 返回结果仍正常
- 结果细胞类型为 `hepatocyte`

可说明：系统支持 ANN 检索与元数据过滤结合。当 ANN 过滤结果不足时，后端会回退到精确过滤检索，保证功能可用。

## 5. 推荐讲解主线

可以按下面这段逻辑组织演示：

```text
本系统面向单细胞基因组数据，支持多角色用户登录、真实 h5ad 数据接入、数据预处理、ANN 索引构建、相似细胞检索、批量检索、可视化分析和诊断报告生成。

本次展示使用老师提供的 liver.h5ad 数据，共 30000 个细胞、32397 个基因。系统完成预处理后生成 PCA 向量，并基于这些向量构建 ANN 索引。

之后用户可以通过 cell_id 发起单次检索，也可以提交批量检索任务。检索结果可以在 UMAP 空间中高亮展示，并可以导出 JSONL/CSV。最后系统可以基于数据集、索引和检索 query_id 生成 PDF/JSON 诊断报告。
```

## 6. 录屏注意事项

- 不建议现场重新完整上传 `liver.h5ad` 并等待预处理，因为文件约 1.4GB，会占用较长时间。
- 如果必须展示上传，建议只展示选择文件和上传入口，然后切换到已处理完成的 `dataset_id = 6` 继续演示。
- 不要删除 `dataset_id = 6`。
- 不要删除 `index_id = 1`。
- 不要删除 admin 用户。
- 生成诊断报告前，先做一次单次检索，确保报告里带有 `query_id`，这样 PDF 内容更完整。
- 如果页面参数显示旧值，刷新页面或使用无痕窗口重新登录。
