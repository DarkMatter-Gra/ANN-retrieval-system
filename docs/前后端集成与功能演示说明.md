## 2. 启动方式

在仓库根目录运行：

```powershell
cd D:\软件工程大作业\ANN-retrieval-system
powershell -ExecutionPolicy Bypass -File .\start.ps1 -NoBrowser
```

访问地址：

- 前端：`http://localhost:4173`
- 后端：`http://localhost:8000`
- Swagger：`http://localhost:8000/docs`

如果前端登录失败，先进入“系统设置”，确认 Base URL 为：

```text
http://localhost:8000/api/v1
```

如果浏览器以前保存过错误地址，也可以清理浏览器 localStorage 后重新登录。

## 4. 推荐录屏顺序

建议使用 `admin / Admin@123` 登录，因为 admin 能看到最多页面，适合完整演示。

### 4.1 登录与工作台

1. 打开 `http://localhost:4173/login`
2. 使用 `admin / Admin@123` 登录
3. 进入工作台，展示角色化导航、数据集数量、索引数量、可用模块入口

可说明：系统支持多角色，不同角色会看到不同导航。

### 4.2 数据集管理

进入“数据集管理”：

- 查看数据集列表
- 点击样例数据集查看详情
- 查看处理日志
- 展示支持上传 `h5ad / mtx / csv`

演示建议：录屏时优先使用已准备好的 smoke 数据，不建议现场上传 `liver.h5ad`，因为真实数据较大，预处理耗时不稳定。

### 4.3 索引管理

进入“索引管理”：

- 查看索引列表
- 选择 `index_id = 1`
- 点击“详情”
- 点击“加载到内存”
- 可展示索引类型、度量、版本、召回率、内存占用

可选：创建新索引。

推荐参数：

```json
{
  "dataset_id": 3,
  "index_name": "demo_flat_l2",
  "index_type": "flat",
  "metric": "l2",
  "params_json": {}
}
```

注意：发布、回滚属于 admin 操作；回滚需要已有目标版本。

### 4.4 单次 ANN 检索

进入“单次 ANN 检索”。

基础检索参数：

```text
数据集 ID: 3
索引 ID: 1
查询类型: cell_id
cell_id: cell_a
Top K: 3
模式: ann
度量: l2
filters JSON: {}
```

预期结果：

```text
cell_a
cell_b
cell_c
```

筛选检索参数：

```json
{"cell_type":["Myeloid"]}
```

预期结果：

```text
cell_d
cell_e
```

可说明：E 模块支持 ANN 检索、精确检索、元数据过滤、维度校验和结果持久化。

### 4.5 检索高亮与可视化

单次检索完成后，页面会得到 `query_id`。

可以：

- 在“单次 ANN 检索”页面点击“加载高亮”
- 或进入“可视化”页面，输入 `dataset_id = 3` 查看 UMAP 嵌入

可说明：系统把检索结果邻居映射到 UMAP 坐标，用于展示 query cell 与近邻细胞的空间关系。

### 4.6 批量检索与任务导出

进入“批量检索”。

参数：

```text
数据集 ID: 3
索引 ID: 1
Top K: 2
模式: ann
导出格式: json
```

queries JSON：

```json
[
  {"query_type":"cell_id","cell_id":"cell_a"},
  {"query_type":"cell_id","cell_id":"cell_d","filters":{"organ":"lung"}}
]
```

提交后：

- 页面显示 `task_id`
- 自动轮询任务状态
- 完成后可下载 JSONL 和 CSV

也可以进入“任务监控”，输入 `task_id`，查询任务并导出。

### 4.7 指标统计

进入“性能指标”：

- 查询搜索指标
- 查看性能监控大盘

可说明：指标页用于观察索引数量、任务状态、延迟趋势、资源状态等。

### 4.8 诊断报告

进入“诊断报告”。

推荐参数：

```text
数据集 ID: 3
索引 ID: 1
包含 QC 报告: 勾选
包含性能报告: 勾选
```

生成后：

- 页面展示报告任务返回 JSON
- 点击“下载 PDF”
- 点击“下载 JSON”

注意：诊断报告页面对角色有限制，`admin / user / auditor` 可访问，`dev` 默认看不到该页面。

### 4.9 用户管理

进入“用户管理”：

- 查看用户列表
- 修改用户角色/状态/配额
- 重置密码
- 删除用户

建议只展示列表和角色修改，不建议在录屏时删除演示账号。

### 4.10 场景化扩展页面

远端 main 新增/升级的场景页可以作为“系统完整性展示”：

- 临床诊断：表型推断、诊断案例对比
- 实验分析：预处理进度、差异基因分析
- API 接入：展示 API 文档/SDK 入口
- 研发工作台：自动调参、基准对比
- 数据库管理：版本、沿袭、租户隔离、公共库同步
- 系统运维：资源监控、告警、自动扩缩容、日志、性能大盘

这些页面大多是轻量业务接口或基于现有数据库记录的展示接口，适合作为“多角色、多场景系统扩展能力”展示；核心 E 模块演示仍建议放在数据集、索引、检索、批量、报告、可视化这条主线。

## 6. 不建议录屏时操作的点

- 不建议现场上传完整 `liver.h5ad`，除非已经预处理完成并确认耗时。
- 不建议删除演示数据集或演示账号。
- 不建议在没有多版本索引时演示回滚。
- 如果用 `dev` 账号，报告页和任务页可能不可见；完整演示优先用 `admin`。
- 如果浏览器保存了旧 Base URL，需要先在系统设置修正为 `http://localhost:8000/api/v1`。

推荐演示主线：

登录 admin → 工作台 → 数据集列表 → 索引详情/加载 → 单次 ANN 检索 → 检索高亮/可视化 → 批量检索与导出 → 性能指标 → 诊断报告 PDF/JSON → 用户管理与多角色能力。
