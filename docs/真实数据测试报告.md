# 真实数据端到端测试报告

## 1. 测试目标

使用老师提供的 `liver.h5ad` 数据包验证当前后端是否可以完成真实数据规模下的核心链路：

- `.h5ad` 数据读取与预处理
- 细胞向量与元数据入库
- HNSW ANN 索引构建
- 单次 ANN 检索
- 单次精确检索
- 元数据过滤检索
- UMAP 可视化数据读取
- 检索结果高亮
- 批量检索与 CSV 导出
- 检索指标查询
- 诊断报告生成

## 2. 测试数据

数据文件位于工作区根目录，未纳入 Git：

```text
../../liver.h5ad
```

数据规模：

| 项目 | 数值 |
|---|---:|
| 文件大小 | 1334.13 MB |

| 基因数 | 32397 |

可用元数据字段包括 `cell_type`、`disease`、`AgeGroup`、`tissue`、`sample_id` 等。

## 3. 测试命令

在 `backend/` 目录执行：

```bash
python scripts/real_liver_backend_test.py ..\..\liver.h5ad
```

脚本会自动完成以下准备工作：

- 设置 `CELERY_TASK_ALWAYS_EAGER=true`
- 创建或复用测试用户 `liver_real_test_user`
- 清理同名旧测试数据集 `liver_real_test`
- 创建数据集记录并执行预处理
- 通过 API 构建 HNSW L2 索引
- 依次调用检索、可视化、导出、指标和报告接口

## 5. 回归测试

在 `backend/` 目录执行：

```powershell
$env:PYTHONPATH='.'; pytest
```

结果：

```text
4 passed, 1 warning
```

说明：直接运行 `pytest` 时，如果当前环境没有把 `backend` 加入模块搜索路径，可能出现 `ModuleNotFoundError: No module named 'app'`。设置 `PYTHONPATH=.` 后可以正常执行。

## 6. 本次发现与处理

真实数据有 69032 个细胞，原预处理逻辑逐条 `db.add()` 会长期保留 ORM 对象，内存压力较高。已将细胞向量和元数据写入改为按 2000 条批量 `bulk_save_objects()`，真实数据测试通过。

## 7. 结论

当前后端已经可以跑通老师提供的真实 `liver.h5ad` 数据，E 模块相关的检索、过滤、可视化高亮、批量导出、指标和诊断报告能力均通过端到端验证。真实数据文件、处理后数据、索引文件、导出文件、报告 JSON/PDF 和本地测试日志属于运行产物，不应提交到 Git。
