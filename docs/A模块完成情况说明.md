# A模块完成情况说明

## 1. 模块职责

我负责的 A 模块主要是前端页面与交互展示，核心职责包括：

- 发起用户登录、注册请求
- 维护登录态和当前用户信息
- 展示数据集列表、详情、预处理日志
- 支持分块上传文件并触发后端预处理流程
- 展示索引列表，并支持创建、加载、上线、回滚索引
- 发起单次检索和批量检索任务
- 轮询任务状态并展示导出链接
- 展示 UMAP / t-SNE 点位和检索高亮结果

本次实现已经从静态页面升级为完整的前端工程，使用 Vite + React + TypeScript 组织页面、状态和 API 交互，并通过根目录 `start.ps1` 实现前后端一键启动，能够以更接近真实项目的方式完成联调与演示。

### 3.2 `src/api.ts`

该文件负责前端与后端的统一请求封装，主要功能包括：

- 统一拼接 API 地址
- 自动附加 `Authorization: Bearer <token>`
- 统一解析后端返回的 `code / message / data` 格式
- 统一处理 JSON 解析错误和 HTTP 错误
- 提供向量输入解析、时间格式化、字节数格式化等工具函数

### 3.3 `src/App.tsx`

该文件负责页面状态、事件绑定和业务流程控制，主要实现：

- 登录后同步当前用户信息

- 分块上传完整链路
- 索引创建、加载、上线和回滚
- 单次检索结果渲染
- 批量检索任务提交与轮询
- 查询高亮点位和可视化渲染
- 本地持久化登录态、数据集、索引和任务信息
- 统一的仪表盘式页面布局

### 3.4 `src/styles.css`

该文件负责整体视觉风格，当前页面采用：

- 深色渐变背景
- 双栏布局
- 卡片化信息展示

## 4. 与其他成员的对接说明

### 4.1 与 B 模块对接

- 前端只通过后端统一 API 访问业务功能
- 登录态通过 `Authorization` 头传递
- 所有展示结果都依赖后端统一响应包装

### 4.2 与 C 模块对接

- 前端不直接解析 `.h5ad`
- 数据上传后，直接调用后端上传与预处理接口
- 检索和可视化结果由后端返回，前端只负责展示

### 4.3 与 D 模块对接

- 前端可创建索引任务并查看索引状态
- 索引构建完成后可直接用于在线检索
- 前端不处理索引内部向量映射，只展示业务结果

### 4.4 与 E 模块对接

- 单次检索结果已经在前端完成展示
- 批量任务、任务状态和导出链接也已具备展示入口
- 可视化高亮结果可直接作为 E 后续演示和评测的展示基础
