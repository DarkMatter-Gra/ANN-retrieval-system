## 1. 系统概述

ANN Retrieval System 是一个面向单细胞基因组数据的近似最近邻（ANN）检索平台。系统支持：

- 上传和管理 h5ad / mtx / csv 等格式的单细胞数据集
- 构建多种 ANN 索引（HNSW、IVF_PQ、Flat）
- 以 cell_id 或浮点向量为查询键执行单次/批量近似最近邻检索
- UMAP / t-SNE 散点图可视化
- 多角色权限隔离、审计日志、性能指标监控

**技术栈**

| 层次 | 技术 |
|------|------|
| 前端 | React 18 + TypeScript + Vite（端口 4173）|
| 后端 | FastAPI + SQLAlchemy + Alembic（端口 8000）|
| 异步任务 | Celery + 本地文件系统队列 |
| 向量库 | FAISS（Flat / IVF_PQ） + hnswlib（HNSW）|
| 数据库 | SQLite（默认）|

## 2. 启动系统

### 2.1 一键启动（推荐）

在项目根目录打开 PowerShell，执行：

```powershell
# 首次使用需先设置执行策略
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

# 启动所有服务
.\start.ps1
```

脚本会自动完成：
1. 检测 Python 3.11/3.12 环境（不满足则自动安装）
2. 创建虚拟环境并安装后端依赖
3. 执行数据库迁移（`alembic upgrade head`）
4. 安装前端 npm 依赖
5. 启动后端 API（新窗口）
6. 启动 Celery Worker（新窗口）
7. 启动前端 Vite 服务（新窗口）
8. 自动打开浏览器 `http://localhost:4173`

| 参数 | 作用 |
|------|------|
| `.\start.ps1` | 正常启动 |
| `.\start.ps1 -ReinstallBackend` | 强制重装 Python 依赖 |
| `.\start.ps1 -ReinstallFrontend` | 强制重装 npm 依赖 |
| `.\start.ps1 -NoBrowser` | 不自动打开浏览器 |

### 2.2 服务地址

| 服务 | 地址 |
|------|------|
| 前端页面 | http://localhost:4173 |
| 后端 API | http://localhost:8000 |
| Swagger 文档 | http://localhost:8000/docs |

## 3. 注册与登录

### 3.0 初始化管理员账号（首次部署）

由于项目仓库不包含 `ann_system.db`（数据库文件由本地启动时自动生成），首次 clone 后数据库为空，前端注册页只能注册 `user` 角色。需要通过脚本一键创建初始管理员：

```bash
cd backend
source .venv/bin/activate          # macOS/Linux
# .venv\Scripts\Activate.ps1       # Windows
python scripts/bootstrap_admin.py
```

执行成功后会输出：

```
created admin user: admin
  username: admin
  password: Admin@123
login at: http://localhost:5173/login
```

随后即可使用 `admin / Admin@123` 在前端登录页登录，进入"用户管理"为其他组员升级角色。

**可选参数**

| 参数 | 作用 | 默认值 |
|------|------|--------|
| `-u, --username` | 自定义管理员用户名 | `admin` |
| `-p, --password` | 自定义管理员密码 | `Admin@123` |
| `-e, --email` | 自定义邮箱 | `admin@example.com` |
| `--reset` | 用户已存在时重置其密码、角色和状态 | — |

示例：

```bash
python scripts/bootstrap_admin.py -u boss -p 'P@ss1234' -e boss@team.com
python scripts/bootstrap_admin.py --reset           # 忘记密码时重置默认 admin
```

> 脚本会自动调用 `Base.metadata.create_all` 建表，无需先跑 `alembic upgrade head`。

### 3.1 注册

访问 `http://localhost:4173/register`，填写：

- **用户名**：唯一标识，不可重复
- **邮箱**：有效邮箱格式
- **密码**：至少 8 位

> 注意：注册时默认角色为 `user`（业务用户）。如需其他角色，须由 admin 在用户管理页修改。

### 3.2 登录

访问 `http://localhost:4173/login`，填写用户名和密码。登录成功后自动跳转至工作台。

### 3.3 退出登录

点击页面右上角用户名旁的**退出**按钮，清除本地 Token 并返回登录页。

1. 在列表中点击目标用户
2. 在"修改角色/状态"面板的"新角色"下拉框中选择目标角色

| locked | 锁定（不可登录）|

1. 在"重置密码"面板填入用户 ID 和新密码（≥8 位）

## 6. 研发人员（dev）

**角色说明**：负责 ANN 索引构建、算法参数调优和性能评估。

### 6.1 可访问功能

工作台 / 研发工作台 / 数据集管理 / 索引管理 / 向量检索 / 批量检索 / 可视化 / 性能指标 / 系统设置

### 6.2 完整使用流程

#### 第一步：上传数据集

路径：`数据集管理`

1. 点击页面底部"上传数据集"区域的"选择文件"
2. 选择 `.h5ad` / `.mtx` / `.csv` 等格式文件
3. 点击"开始上传"，进度条实时显示分块上传进度
4. 上传完成后记录返回的**数据集 ID**（后续所有操作均需此 ID）

**查看数据集状态**
- 在列表中点击数据集行，右侧面板显示详细信息和处理日志
- QC 状态和预处理状态需通过后端任务处理后更新

#### 第二步：创建索引

路径：`索引管理`

1. 在"创建索引任务"面板填写：

   | 字段 | 说明 |
   |------|------|
   | 数据集 ID | 第一步中的数据集 ID（须预处理完成）|
   | 索引名称 | 自定义名称 |
   | 索引类型 | `flat`（精确）/ `ivf_pq`（量化压缩）/ `hnsw`（图索引）|
   | 度量 | `l2`（欧氏距离）/ `cosine`（余弦）/ `ip`（内积）|
   | 参数 JSON | 算法超参，见下表 |

   **索引参数参考**

   | 索引类型 | 参数示例 |
   |----------|---------|
   | flat | `{}` |
   | ivf_pq | `{"nlist": 128, "m": 8, "nbits": 8}` |
   | hnsw | `{"M": 16, "ef_construction": 200, "ef": 64}` |

2. 点击"创建索引任务"，后台异步构建
3. 在列表中刷新，等待"状态"变为**已完成（done）**
4. 记录**索引 ID**

> 数据集的 `preprocess_status` 必须为 `done` 才能创建索引。

#### 第三步：加载索引到内存

选中已完成的索引，点击"加载到内存"，使其可以被检索服务调用。

#### 第四步：执行向量检索

路径：`向量检索`

1. 填写数据集 ID 和索引 ID
2. 选择查询类型：
   - **cell_id**：输入现有细胞 ID（如 `cell_0001`），系统自动取对应向量
   - **vector**：直接输入浮点向量（逗号分隔，如 `0.1, 0.5, -0.3, ...`）
3. 设置 Top K（返回最相似的 K 个结果，范围 1-1000）
4. 选择模式：
   - **ann**：近似最近邻（快速）
   - **exact**：精确暴力搜索（精确但慢）
5. 点击"执行检索"

**检索结果说明**

| 字段 | 含义 |
|------|------|
| rank | 排名 |
| cell_id | 目标细胞 ID |
| distance | 向量距离（越小越相似）|
| score | 相似度分数 = 1/(1+distance)（越大越相似）|
| cell_type | 细胞类型标注（若有）|
| organ | 组织/器官（若有）|

#### 第五步：查看性能指标

路径：`性能指标`

查看历史检索任务的延迟分布、召回率统计等指标，用于评估和调优索引参数。

### 6.3 批量检索

路径：`批量检索`

当需要对大量 cell_id 或向量批量查询时：

1. 填写数据集 ID、索引 ID
2. 在 queries JSON 输入框填写查询列表：

```json
[
  {"query_type": "cell_id", "cell_id": "cell_0001"},
  {"query_type": "cell_id", "cell_id": "cell_0002"},
  {"query_type": "vector", "vector": [0.1, 0.5, -0.3]}
]
```

3. 设置 Top K、模式、导出格式（json / csv）
4. 点击"提交批量任务"，系统后台异步处理
5. 页面每 3 秒自动刷新任务状态和进度条
6. 完成后点击"下载结果"获取文件

## 8. 服务账号（service）

**角色说明**：专为程序化 API 调用设计，通过 JWT Token 对接外部系统或自动化脚本。

### 8.1 可访问功能

工作台 / API 接入 / 向量检索 / 批量检索 / 任务监控 / 系统设置

### 8.2 获取 Token

通过 API 登录获取 Bearer Token：

```bash
curl -X POST http://localhost:8000/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username": "your_service_account", "password": "your_password"}'
```

返回示例：
```json
{"code": 0, "data": {"access_token": "eyJ...", "token_type": "Bearer"}}
```

后续请求在 Header 中携带：
```
Authorization: Bearer eyJ...
```

### 8.3 程序化调用主要接口

**单次检索**
```bash
POST /api/v1/search
{
  "dataset_id": 1,
  "index_id": 1,
  "query_type": "cell_id",
  "cell_id": "cell_0001",
  "top_k": 10,
  "mode": "ann"
}
```

**提交批量检索任务**
```bash
POST /api/v1/batch-search
{
  "dataset_id": 1,
  "index_id": 1,
  "top_k": 10,
  "export_format": "json",
  "queries": [
    {"query_type": "cell_id", "cell_id": "cell_0001"},
    {"query_type": "vector", "vector": [0.1, 0.5, ...]}
  ]
}
```

**查询任务状态**
```bash
GET /api/v1/tasks/{task_id}
```

**下载任务结果**
```bash
GET /api/v1/tasks/{task_id}/export?format=json
GET /api/v1/tasks/{task_id}/export?format=csv
```

完整 API 文档见：`http://localhost:8000/docs`

### 8.4 任务监控（页面）

路径：`任务监控`

- 输入 task_id 查询任务状态
- 查看任务进度、错误信息
- 下载 JSON / CSV 格式结果

## 11. 通用功能说明

### 11.1 工作台

路径：`工作台`（登录后默认页）

| 角色 | 工作台标题 |
|------|-----------|
| admin | 系统管理员工作台 |
| dev | 研发工程师工作台 |
| user | 业务用户工作台 |
| service | API 服务账号 |
| readonly | 只读观察账号 |
| auditor | 审计评估账号 |

### 11.2 向量检索详解

路径：`向量检索`

**参数说明**

| 参数 | 说明 | 默认值 |
|------|------|--------|
| 数据集 ID | 目标数据集的 ID | — |
| 索引 ID | 已完成构建的索引 ID | — |
| 查询类型 | `cell_id`（现有细胞）/ `vector`（任意向量）| cell_id |
| Top K | 返回最相似的结果数量 | 10 |
| 模式 | `ann`（近似，快）/ `exact`（精确，慢）| ann |
| 度量 | `l2`（欧式）/ `cosine`（余弦）/ `ip`（内积）| l2 |
| ef_search | HNSW 索引的查询精度参数，越大越精确但越慢 | — |

**获取索引 ID 的方法**

- 联系管理员或研发人员提供可用的索引 ID
- 或在"索引管理"页面（需要 admin / dev 角色）查看

**检索高亮**

检索完成后，页面会保存 `query_id`。在"检索高亮"面板输入 `query_id` 点击"加载高亮"，可获取结果点位信息，与可视化页面联动。

## 12. 常见问题

**Q: 登录后页面不断闪烁并提示 Failed to fetch**

原因：后端服务未启动，或前端 Base URL 配置错误。

解决方案：
1. 确认 `.\start.ps1` 已执行且所有窗口正常运行
2. 进入"系统设置"，确认 Base URL 为 `http://localhost:8000/api/v1`
3. 点击"健康检查"验证连接

---

**Q: 向量检索提示 index not found**

需同时满足三个条件：
1. 索引 ID 存在，且属于填写的数据集 ID
2. 索引构建状态为 `done`（已完成）
3. 使用自己创建的索引（非 admin 时）

解决方案：到`索引管理`页（需 admin/dev 权限）查看可用索引，或联系管理员提供。

---

**Q: 索引构建一直处于 pending / running**

Celery Worker 可能未启动。检查后端目录下是否有"run-worker"窗口正常运行，或重新执行 `.\start.ps1`。

---

**Q: 上传数据集失败（Failed to fetch）**

1. 确认后端服务正常（健康检查通过）
2. 确认磁盘有足够空间（数据文件会存储在后端 `data/` 目录）
3. 确认文件格式为 `.h5ad` / `.mtx` / `.csv` / `.h5` / `.loom`

---

**Q: 注册后无法访问某些功能**

注册默认角色为 `user`。如需 `dev`、`admin` 等角色，请联系管理员在"用户管理"页修改角色。

如果是首次部署、还没有任何 admin 账号，请先按 [3.0 初始化管理员账号](#30-初始化管理员账号首次部署) 跑一次 `python scripts/bootstrap_admin.py`。

---

**Q: 批量检索任务长时间无进度**

批量任务由 Celery Worker 异步处理。确认 Worker 正常运行，并等待状态从 `pending` → `running` → `done`。页面每 3 秒自动刷新一次。
